Class DataTypeServices

A service that knows about data types providing tools for:

  • Identifying them
  • Converting them into something better suited for comparisons and formatting
  • Formatting them for the tokens of error messages
  • Comparing them It works in conjunction with the DataType Lookup Keys Lookup Keys.

Formatting supports localization, keeping a list of one or more IDataTypeLocalizedFormatters, for all supported cultures. As you set it up, you must supply a list of one or more CultureConfig objects. Each identifies a culture that you support and a fallback culture when the desired culture didn't support the Format.

CompareValues

  • CompareValues(value1, value2, lookupKey1, lookupKey2): ComparersResult

  • Compares two same-type values to see if they are equal or not. It can return Equals and NotEquals for types that make no sense to support greater than and less than. Otherwise it returns Equals, LessThan, or GreaterThan. Expect exceptions when invalid values are supplied. It identifies the ComparerHandler function. If either value is null, it will either return Equals (both null) or Undetermined.

    Parameters

    • value1: any
    • value2: any
    • lookupKey1: null | string

      Identifies the IDataTypeConverter and/or ComparerHandler function to use together with value1. If null, the native data type of the value will be converted to a lookupKey when String, Number, Boolean, Date object, or any IDataTypeIdentifier that you have registered with the DataTypeServices.

    • lookupKey2: null | string

      Same idea as lookupKey1 but for value2. This value will not be used to find a ComparerHandler.

    Returns ComparersResult
