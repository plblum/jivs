ValueHostId

ValueHostId: null | string

One source for the value to evaluate. By design, Condition.Evaluate takes a valueHost object, allowing the caller to simply pass in the value. Leave this null to use that valueHost object.

Assign this to a ValueHostId if you want to have it looked up in the ValueHostsManager.GetValueHost.

Typically leave InputValidator.ConditionDescriptor.ValueHostId null because Condition.Evaluate is passed the correct valueHost. However, InputValidator.EnablerDescriptor needs it assigned. Same with any Condition that is a child of another, like in MultiConditions.

Many conditions need two or more sources for values. They are expected to create more ValueHostId properties in their ConditionDescriptor, where the remaining Properties are identified.
