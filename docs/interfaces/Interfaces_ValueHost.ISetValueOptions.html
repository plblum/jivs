Optional ConversionErrorTokenValue

ConversionErrorTokenValue?: string

When converting the input field/element value to native and there is an error it should be supplied here. Only meaningful when State.Value is undefined. It can be displayed as part of the DataTypeCheckCondition's error message token {ConversionError}. Cleared when setting the value without an error.

  • Defined in src/Interfaces/ValueHost.ts:153

Optional Reset

Reset?: boolean

Reset the field's changed and validation states as if the field has never been edited. It effectively sets ValueHost.IsChanged to false and calls InputValueHost.ClearValidation.

  • Defined in src/Interfaces/ValueHost.ts:144

Optional SkipValueChangedCallback

SkipValueChangedCallback?: boolean

If you have setup the OnValueChanged callback, it automatically is run when a value is changed. Use this to skip that callback. Callback is skipped when true. When null, it is the same as false.
